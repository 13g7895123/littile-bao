## 一、目標與範圍

### 1.1 業務目標

將儀表板（`gui.py` + `engine.py`）從「**全模擬資料**」升級為「**接富邦真實 API**」，達成：

- 6 張統計卡顯示真實帳戶損益、報酬率、可用額度

- 支援真實下單、改價、改量、刪單
- 開盤前自動掃描動態選股清單，取代 `MOCK_STOCKS`

### 1.2 非目標（本期不做）

- GUI 視覺風格大幅重構（沿用現有 PySide6 介面）
- K 線回測、技術指標分析
- 多帳號跨機構整合
- 雲端部署（暫定本機 Windows / Linux 執行）

## 三、目標架構設計

### 3.1 高階分層圖

```
┌─────────────────────────────────────────────────────────────┐
│                     PySide6 GUI（gui.py）                    │
│  統計卡 6 / 監控表 / 持倉表 / 委託表 / 成交表                  │
└──────────────┬──────────────────────────┬───────────────────┘
               │ Qt Signal                │ Qt Signal
┌──────────────▼──────────────────────────▼───────────────────┐
│              TradingEngine（engine.py 重構）                 │
│   策略狀態機（F1~F8）+ 進場 / 出場決策 + 統計聚合              │
└──┬──────────┬──────────────┬──────────────┬─────────────────┘
   │ Push     │ Push         │ Poll         │ Order
┌──▼──┐    ┌──▼──┐        ┌──▼──┐        ┌──▼─────┐
│ RT  │    │Order│        │Acct │        │ Order  │
│Feed │    │Event│        │Poll │        │ Sender │
└──┬──┘    └──┬──┘        └──┬──┘        └──┬─────┘
   │          │              │              │
┌──▼──────────▼──────────────▼──────────────▼─────┐
│              FubonAdapter（新增模組）             │
│   封裝 Fubon Neo SDK；單例；憑證 / 連線 / 訂閱   │
└──────────────────────┬──────────────────────────┘
                       │
                  Fubon Neo SDK
```

### 3.2 設計原則

1. **單一資料源**：所有真實資料皆透過 `FubonAdapter` 取得，禁止散落於 GUI
2. **訊號驅動**：`engine` ↔ `gui` 透過 Qt Signal 通訊；`adapter` ↔ `engine` 透過 callback 或 `queue.Queue`
3. **可降級**：保留 `MOCK_MODE=true` 開關，無憑證時仍可跑模擬（便於 Demo / 開發）
4. **執行緒安全**：Tick / Order callback 由 SDK 子執行緒呼叫，所有跨執行緒寫入需 lock 或 thread-safe queue
5. **失效復原**：WebSocket 斷線自動重連、補拉缺失資料

## 七、關鍵實作細節

### 7.1 執行緒模型

| 執行緒 | 工作 |
|---|---|
| **GUI 主執行緒** | PySide6 事件迴圈，僅消費 Signal |
| **Engine 執行緒** | 既有 `_loop`，現改為 event-driven（從 `queue.Queue` 取 TickEvent） |
| **SDK 行情執行緒** | RealtimeClient WebSocket 接收，callback 入 `queue` |
| **SDK 委託執行緒** | `on_order` / `on_filled` callback，發 Qt signal（須用 `QMetaObject.invokeMethod` 或 `pyqtSignal`） |
| **Account Poll Timer** | `QTimer.singleShot(10000)`，主執行緒呼叫 → 子執行緒查詢 |

### 7.2 1 秒成交量滑動視窗

```python
from collections import deque
from time import monotonic

class VolumeWindow:
    def __init__(self, window_sec: float = 1.0):
        self.window = window_sec
        self.buf: deque[tuple[float, int]] = deque()  # (ts, volume)

    def add(self, volume: int) -> int:
        now = monotonic()
        self.buf.append((now, volume))
        while self.buf and now - self.buf[0][0] > self.window:
            self.buf.popleft()
        return sum(v for _, v in self.buf)
```

### 7.3 委賣張數（漲停板）判斷

```python
def calc_limit_up_ask(book: BookData, limit_up_price: Decimal) -> int:
    if not book.ask:
        return 0
    if Decimal(str(book.ask[0].price)) == limit_up_price:
        return book.ask[0].volume
    return 0  # 板已打開或尚未漲停
```

### 7.4 損益計算（含手續費 / 稅）

```python
FEE_RATE = Decimal("0.001425")
TAX_RATE = Decimal("0.003")
MIN_FEE  = Decimal("20")

def calc_fee(amount: Decimal) -> Decimal:
    return max(amount * FEE_RATE, MIN_FEE).quantize(Decimal("1"))

def realized_pnl(buy_price, sell_price, qty_lots) -> Decimal:
    buy_amt  = buy_price  * qty_lots * 1000
    sell_amt = sell_price * qty_lots * 1000
    fee_buy  = calc_fee(buy_amt)
    fee_sell = calc_fee(sell_amt)
    tax      = (sell_amt * TAX_RATE).quantize(Decimal("1"))
    return sell_amt - buy_amt - fee_buy - fee_sell - tax
```

## 十、風險與相依性

| 風險 | 影響 | 緩解 |
|---|---|---|
| Fubon SDK Wheel 僅支援 manylinux 與 Win | 部署環境受限 | 鎖定 Python 3.11；提供 Dockerfile |
| WebSocket 斷線時錯失漲停瞬間 | 策略誤判 | 重連後比對最新 `books`；補拉 `intraday/quote` |
| 憑證到期 | 全部功能停擺 | 啟動時檢查到期日，30 天前 GUI 警示 |
| 下單頻率超過券商限制 | 委託被拒 | `engine.py` 加入 token bucket 限流 |
| Callback 跨執行緒寫 GUI | 崩潰 | 強制透過 `pyqtSignal` |
| 模擬模式與真實模式行為不一致 | 上線後 bug | 兩者共用同一份 `engine` 邏輯，僅 Adapter 不同 |
