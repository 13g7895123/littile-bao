## GUI 主要對應區塊

| GUI 區塊 | 用途 |
|---|---|
| `儀表板` | 左側策略設定；右側看即時監控、事件日誌、持倉/委託/成交預覽 |
| `策略設定` | 與儀表板左側同一份設定，但改成全頁顯示 |
| `委託/成交` | 看買入、賣出、取消委託實際送單與成交結果 |
| `持倉部位` | 看目前部位、成本、現價、損益 |
| `事件日誌` | 看策略略過、進場、賣出、特殊股排除等訊息 |
| `決策明細` | 看更細的策略判斷事件 |

## 賣出條件對應

## 補充說明

### 1. 目前哪些條件是「有參數，但沒有獨立 on/off 開關」

| 條件 | 現況 |
|---|---|
| 漲停打開賣出 | 有參數欄位，但目前 GUI 收集設定時 `f4_enabled` 固定為 `True` |
| 1 秒爆量賣出 | 有張數 / 比例欄位，但目前 GUI 收集設定時 `f5_enabled` 固定為 `True` |
| 委託排隊中遇爆量取消 | 有 `取消委託 / 保留委託` 行為選擇，但目前 GUI 收集設定時 `f6_enabled` 固定為 `True` |
| 每日最大交易檔數 | 有數值欄位，但目前 GUI 收集設定時 `f13_enabled` 固定為 `True` |

如果你要的是「每一條策略都能單獨勾選開 / 關」，那目前 GUI 還少幾個獨立 checkbox。

### 2. 目前最常用的觀察位置

| 想看什麼 | 建議看哪裡 |
|---|---|
| 哪些股票正在等進場、被略過、已封鎖 | `儀表板 > 即時監控` 的 `狀態 / 動作` 欄 |
| 為什麼這檔沒有買 | `事件日誌` 或 `決策明細` |
| 買單、賣單、取消單是否真的送出 | `委託/成交` |
| 目前還持有哪幾檔 | `持倉部位` |

### 3. 依你這份需求，和 GUI 最接近的設定區塊

若只看操作面，最主要會用到的是 `策略設定` 頁這幾段：
- `市場選擇`
- `交易設定`
- `買入策略 > 時間設定`
- `買入策略 > 進場條件`
- `排除條件`
- `賣出策略`

## 結論

以你這份需求來看，現有 GUI 已經能直接對應大部分核心條件，特別是：
- 市場選擇
- 每檔金額
- 價格區間
- 買入時間
- 第一根 / 第二根
- 特殊股排除
- 漲停打開賣出
- 1 秒爆量賣出
- 每日最大買入檔數

目前比較明顯還不夠直覺的地方有三個：
- `8% 開始偵測` 沒有獨立 GUI 欄位
- `賣出時間 09:00-13:25` 沒有獨立 GUI 欄位
- 幾條賣出 / 取消規則雖然有參數，但沒有獨立 `開 / 關` checkbox
