# 儀表板分頁測試方案

## 測試目標
驗證儀表板保留原本總覽長相與功能，包含統計卡、即時監控、事件日誌、持倉、委託與成交小區塊；同時確認策略設定改在 `策略設定` 分頁內，其他分頁顯示各自對應資訊的完整版本。

## 自動化測試
執行：
```bash
cd build_win/src
PYTHONDONTWRITEBYTECODE=1 QT_QPA_PLATFORM=offscreen python3 -m unittest tests.test_gui_tabs -v
```

覆蓋項目：
- 主體區只保留分頁內容容器，沒有常駐左側策略設定欄。
- `策略設定` 分頁內含策略設定欄位，例如 `per_stock_amount`。
- `儀表板` 分頁保留原本總覽區塊：統計卡、即時監控、事件日誌、持倉、委託、成交。
- `委託/成交`、`持倉部位`、`事件日誌` 分頁各自持有自己的主要表格或日誌元件。
- 委託狀態更新、成交記錄更新、事件日誌追加會同時反映在儀表板小區塊與對應完整分頁，且切換分頁不會重複寫入。

## 回歸測試
執行既有單元測試：
```bash
cd build_win/src
PYTHONDONTWRITEBYTECODE=1 QT_QPA_PLATFORM=offscreen python3 -m unittest discover -s tests -v
```

通過標準：所有既有 `tests/` 單元測試與新增 GUI 分頁測試皆通過。

## 手動驗收
執行桌面程式後檢查：
```bash
cd build_win/src
QT_QPA_PLATFORM=xcb python3 main.py
```

驗收項目：
- 開啟後 `儀表板` 維持原本總覽畫面：統計卡、即時監控、事件日誌、持倉、委託、成交，不顯示常駐左側策略設定。
- 點 `策略設定` 後才顯示策略設定內容與匯入、匯出、重置、儲存按鈕。
- 點 `委託/成交`、`持倉部位`、`事件日誌` 後，各自顯示比儀表板小區塊更完整的對應內容。
- 切換分頁時狀態列與標題列維持正常，沒有控制項重疊或空白錯位。

## 已知限制
本方案以 PyQt offscreen 自動化確認結構與資料路徑；實際視覺排版仍建議在有桌面環境的 Linux/Windows 上人工確認一次。
