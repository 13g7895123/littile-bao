## 2. 完全沒有 GUI 欄位

### 2.1 「8% 以上開始偵測」門檻

- 需求：上市上櫃股票漲幅達 8% 以上才開始偵測候選。
- 現況：`gui_pages.py:228` 的 `limit_up_detection_mode` 是「鎖板判定模式」下拉，控制的是「怎樣算鎖漲停」，不是「漲幅多少開始進入候選池」。
- 全專案搜尋 `detection_threshold` / `trigger_pct` / `0.08` 皆無對應欄位。
- 引擎內目前是依鎖漲停 / 候選條件 + 行情判斷，並非單一百分比門檻。

### 建議修法

在「買入策略 > 進場條件」新增 `偵測啟動漲幅 >= X %` 欄位，預設 8。需要在 `config.py` 增加欄位（例如 `detect_min_change_pct`），並於候選過濾階段（snapshot_cache 或 engine 候選 hook）使用。

### 2.2 賣出時間 `09:00-13:25`

- 需求：賣出限定時段內才作動。
- 現況：買入有 `start_time / entry_before_time`，賣出**沒有對應欄位**。F4 / F5 / F6 觸發完全依盤中事件，不受時段限制。
- 搜尋 `sell_start_time` / `sell_end_time` / `exit_start_time` 皆無命中。

### 建議修法

`config.py` 新增 `sell_start_time / sell_end_time`，預設 `09:00 / 13:25`；引擎在 F4/F5 觸發前加時段檢查。GUI 在「賣出策略」section 開頭增加兩個時間欄位。

## 3. 文案 / 體驗待調整（非缺功能，但與需求語意不符）

### 3.1 漲停委賣張數需求是固定 0，但 GUI 為可調門檻

- 需求：「漲停委賣價張數為 0 買入」（張數固定為 0 才買）。
- 現況：[gui.py:1444](/home/jarvis/project/bonus/03_littile-bao/build_win/src/gui.py:1444) 收集 `ask_queue_threshold`，判斷在 [gui.py:3358](/home/jarvis/project/bonus/03_littile-bao/build_win/src/gui.py:3358) 為 `ask_qty >= ask_queue_threshold`，使用者需自行填 `1` 才近似「0 張才買」。
- 文案上看起來像「小於幾張」，但實際語意需用戶自行換算。

### 建議修法

兩種做法擇一：
1. 維持門檻可調，但把文案改成「漲停委賣 < X 張即視為可進場」，預設 1。
2. 改成 checkbox「漲停委賣張數為 0 才買」＋ 進階模式才暴露門檻欄位。

### 3.2 三項排除股勾選實際上是綁在一起

- 需求：「限當沖股 / 處置股 / 注意股都不買」。
- GUI：[gui_pages.py](/home/jarvis/project/bonus/03_littile-bao/build_win/src/gui_pages.py) 有 `excl_disposal / excl_attention / excl_daytrade` 三個 checkbox。
- 但收集設定時三者合併為單一 `f11_enabled`（[gui.py:1380-1382](/home/jarvis/project/bonus/03_littile-bao/build_win/src/gui.py:1380)、[gui.py:1467](/home/jarvis/project/bonus/03_littile-bao/build_win/src/gui.py:1467)），等於「任一勾即三者全開、三者全關才關」，並非各自獨立。

### 建議修法

`config.py` 拆成 `exclude_disposal / exclude_attention / exclude_day_trade_restricted` 三個布林（candidate `Criteria` 本來就支援，見 [gui.py:919-921](/home/jarvis/project/bonus/03_littile-bao/build_win/src/gui.py:919)），引擎側使用個別旗標即可。

下列項目程式碼已符合文件描述，不需處理：
