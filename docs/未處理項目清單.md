## 二、各項詳細說明

### 1️⃣ F4 容忍度（檔位 / 秒數）

**現況**：[engine.py](build_win/src/engine.py) `_tick()` 出場分支
```python
if cfg.f4_enabled and has_been_at_limit_today \
        and not state.is_at_limit_up:
    reason = "委買漲停，市場打開，市價出場"
```
只要 `ask[0]` 不在漲停價，立即觸發。

**問題**：板被吃開 1 檔 → 立刻補回 → 被誤殺出場。

**建議實作**：
```python
# config.py
f4_tolerance_ticks: int = 0      # 0 = 立即賣
f4_open_seconds: float = 0.0     # 板打開持續 N 秒才賣
```
```python
# engine.py（StockState 新增）
opened_since: Optional[float] = None

# _tick 內
limit_up = Decimal(str(info.limit_up))
tick = tick_size(limit_up)
threshold = limit_up - tick * Decimal(cfg.f4_tolerance_ticks)
opened = state.ask0_price is not None and state.ask0_price < threshold
if opened:
    state.opened_since = state.opened_since or now
    if now - state.opened_since >= cfg.f4_open_seconds:
        reason = "..."
else:
    state.opened_since = None
```

需要 `tick_size()` 工具函式（依台股價位表）：
- ≤ 10：0.01
- ≤ 50：0.05
- ≤ 100：0.10
- ≤ 500：0.50
- ≤ 1000：1.00
- > 1000：5.00

**估時**：含測試約 0.5 工作日。

### 2️⃣ F1 ↔ F10 互斥開關

**現況**：兩者皆 enabled 時是 AND 關係。

| 條件 | F1 期望 | F10 期望 |
|---|---|---|
| 委賣張數 | **少** (`< ask_queue_threshold`) | — |
| 1 秒成交量 | — | **多** (`≥ entry_volume_confirm`) |

**問題**：早盤搶漲停（F1）與消化量進場（F10）邏輯方向相反，使用者擇一啟用。

**建議實作**：
```python
# config.py
f1_f10_exclusive: bool = True
```
```python
# engine.start()
if cfg.f1_f10_exclusive and cfg.f1_enabled and cfg.f10_enabled:
    self.on_log("WARN", "F1 與 F10 同時啟用，互斥模式：以 F10 為主，F1 自動關閉")
    cfg.f1_enabled = False    # 或反之，視 GUI 提示讓使用者選
```

GUI 端可加 RadioButton 取代 CheckBox，或 enable/disable 對方欄位。

**估時**：含 GUI 約 0.5 工作日。

### 3️⃣ 漲停消化量進場（新功能）

> 用戶描述：「委賣在漲停，還在消化量，成交幾張我就要進場（可關掉功能 1）」

**這不是現有的 F13**（F13 是當日成交檔數上限）。是一個**全新進場條件**。

**建議命名**：避免和 F13 混淆，用 `f_consume` 命名。

```python
# config.py
f_consume_enabled: bool = False
consume_qty_threshold: int = 499   # 漲停期間累計成交達 N 張即進場
```

**狀態欄位**：
```python
# StockState
limit_up_consumed: int = 0   # 自首次漲停起累計的成交量（張）
```

**邏輯**：
- `_on_tick`：當 `state.is_at_limit_up == True` 時，把 `ev.volume` 累加到 `state.limit_up_consumed`
- `_on_book`：板打開時 `state.limit_up_consumed = 0` 重置
- `_tick` 進場分支：
  ```python
  if cfg.f_consume_enabled:
      if state.limit_up_consumed < cfg.consume_qty_threshold:
          return
  ```

**估時**：含測試約 1 工作日。

### 4️⃣ F12 「開盤就漲停」獨立開關

**現況**：F12 只在 `open_limit_up && sold_today` 雙條件下封鎖。沒有「我直接不要追開盤就漲停的股票」這個選項。

**建議實作**：
```python
# config.py
f_open_limitup_block: bool = False   # True = 開盤即漲停一律不追
```
```python
# engine._tick 進場前
if cfg.f_open_limitup_block and info.open_limit_up:
    return
```

**估時**：< 0.5 工作日。
